# Agent Task: Add Arrival Confirmation System

**Committed on:** January 3, 2026 at 8:05 AM

## Overview

Require sustained detection (10s, 30% hit rate) before confirming arrival. Recordings and snapshot start immediately on first detection, but use `.tmp` suffix. On confirmation success: rename files, send notification. On confirmation failure: keep `.tmp` files, reset to ABSENT, no notification.

### Modify `_handle_event()` for ARRIVED events:

Instead of immediately handling ARRIVED:

1. Set `arrival_pending = True`
2. Set `arrival_pending_start = now`
3. Reset counters: `arrival_detection_count = 1`, `arrival_frame_count = 1`
4. Save snapshot with `temp=True`, store path in `pending_snapshot_path`
5. Start recordings (they already use `.tmp` during recording)
6. Do NOT call `event_handler.handle_event(ARRIVED, ...)` yet
7. Do NOT send notification yet

### Modify `process_frame()`:

After detection, before state machine update, if `arrival_pending`:

```python
if self.arrival_pending:
    self.arrival_frame_count += 1
    if falcon_detected:
        self.arrival_detection_count += 1

    elapsed = (now - self.arrival_pending_start).total_seconds()
    if elapsed >= self.arrival_confirmation_seconds:
        ratio = self.arrival_detection_count / self.arrival_frame_count

        if ratio >= self.arrival_confirmation_ratio:
            # SUCCESS
            self._confirm_arrival()
        else:
            # FAILURE
            self._cancel_arrival(ratio)
```

### Add confirmation success method:

```python
def _confirm_arrival(self) -> None:
    """Arrival confirmed - rename files, send notification."""
    logger.event(f"✅ Arrival confirmed: {self.arrival_detection_count}/{self.arrival_frame_count} frames")

    # Rename snapshot
    if self.pending_snapshot_path and self.pending_snapshot_path.exists():
        final_path = Path(str(self.pending_snapshot_path).replace('.tmp', ''))
        self.pending_snapshot_path.rename(final_path)

    # Rename recordings (arrival clip, visit recorder)
    self.arrival_clip_recorder.rename_to_final()
    self.visit_recorder.rename_to_final()

    # NOW send notification and emit event
    self.event_handler.handle_event(FalconEvent.ARRIVED, self.arrival_pending_start, {})

    # Reset pending state
    self.arrival_pending = False
    self.arrival_pending_start = None
    self.pending_snapshot_path = None
```

### Add confirmation failure method:

```python
def _cancel_arrival(self, ratio: float) -> None:
    """Arrival not confirmed - keep .tmp files, reset state."""
    logger.warning(
        f"⚠️ Arrival not confirmed: {ratio:.1%} detection rate "
        f"({self.arrival_detection_count}/{self.arrival_frame_count} frames)"
    )

    # Keep .tmp files (do not delete - useful for debugging)

    # Stop recordings without renaming
    self.arrival_clip_recorder.stop()
    self.visit_recorder.stop_recording(datetime.now())  # Returns but we ignore

    # Reset state machine
    self.state_machine.reset_to_absent()

    # Reset pending state
    self.arrival_pending = False
    self.arrival_pending_start = None
    self.pending_snapshot_path = None
```

## Test Scenarios

1. **Real arrival**: Sustained detections → files renamed to final, notification sent
2. **False arrival**: 1-2 detections then nothing → `.tmp` files kept, no notification, state reset
3. **Departure during confirmation**: Bird leaves before 10s → treat as failed, no departure notification either
