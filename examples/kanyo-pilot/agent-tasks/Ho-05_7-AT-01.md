# Agent Task: Fix Departure Clip Timing

**Committed on:** December 30, 2025 at 7:29 PM

## Problem
Departure clips start AFTER the falcon has already left. The clip misses the actual departure moment.

## Example (Real Data)

**Visit at 05:52:22 stream local:**
- Visit recording started: 05:52:22
- Actual departure (visible in video): 06:12:44
- Expected last_detection_offset: 06:12:44 - 05:52:22 = **1222 seconds**

**What the code calculated:**
```
Last detection at offset 1274.2s, extracting 1244.2s + 45s
```
- 05:52:22 + 1244.2s = 06:13:06 ← clip starts here
- But falcon left at 06:12:44 ← **22 seconds earlier!**

**Result:** Departure clip shows empty nest, falcon already gone.

## The Bug
`last_detection_offset` is ~52 seconds too high (1274 vs ~1222).

Possible causes:
1. Exit timeout (90s) being added to the offset
2. Stream outages inflating frame count
3. Frame count ÷ FPS not matching wall clock time
4. `last_detection_time` not being updated during visit

## Files to Check
- `src/kanyo/detection/buffer_clip_manager.py` — How is `last_detection_offset` calculated?
- `src/kanyo/detection/falcon_state.py` — Is `last_detection_time` updated on each detection?
- `src/kanyo/__main__.py` or `buffer_monitor.py` — Where is offset passed to clip manager?

## Expected Behavior
1. Track `last_detection_time` = wall clock time of most recent bird detection
2. On departure: `last_detection_offset` = `last_detection_time` - `visit_start_time`
3. Departure clip starts at `last_detection_offset - 30s` into visit recording
4. Clip shows ~30s before bird leaves, ~15s after = captures actual departure

## Debug Logging
Add to detection loop:
```python
if bird_detected:
    logger.debug(f"Bird detected at {now}, updating last_detection_time")
    self.last_detection_time = now
```

Add to departure handler:
```python
logger.debug(f"Visit started: {visit_start_time}")
logger.debug(f"Last detection: {last_detection_time}")
logger.debug(f"Calculated offset: {last_detection_time - visit_start_time} seconds")
```
