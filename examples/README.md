# Examples

Evidence from practice — annotated project arcs demonstrating the Ho System in use.

## Kanyō Pilot

The first application of the Ho System: a production computer vision system built by a non-developer in 6 weeks.

See `kanyo-pilot/` for the full arc and `learning-process.md` for meta-reflections.

## pālana Pilot — Continuity Discipline

The first *autonomous* Ho System build: the agent authors the whole Kamae chain and executes it, with the practitioner in only for UI/UX sessions. `palana-autonomous/continuity-discipline.md` is a practice report on how it maintained continuity across sessions with no human carrying the thread — a four-layer system (working-memory handoff, public build record, per-ho Reflect, an ntfy heartbeat), written as a repeatable process, with proposed additions the framework could adopt for autonomous or long-running builds.

## sage-zfs — Encoded Environment Portability

The first Ho System build run on a *non-Claude* agent (Codex / gpt-5.6-sol). `sage-zfs-codex/encoded-environment-portability.md` is a practice report on the vendor crossing: one AGENTS.md file pointed the agent at the same skills and discipline documents Claude uses, and the output was a conformant Kamae chain — including a correctly banked sealed decision and a declared compression — plus the practitioner's first read of the new model's grain. Evidence for the encoded-environment thesis at its strongest bar: the artifacts carry the discipline, whoever is reading them.
